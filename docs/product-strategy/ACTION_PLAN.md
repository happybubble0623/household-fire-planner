## 1. Strategic thesis

**Don't try to out-simulate cFIREsim and FIREproof — you'll lose on simulation depth and 14 years of brand.** Instead, win a different segment they structurally under-serve: **beginners + pre-65 healthcare + households**, served as one free, private, plain-language plan. That wedge is discovered almost entirely through **long-tail SEO** (5k/mo is the sum of ~30–50 modest pages, not one famous tool). Add *just enough* credibility — an honest sequence-of-returns caveat now, and a real success-rate % later — so serious users don't dismiss it as a toy, but never get drawn into the optimizer arms race. **Keep monetization low-maintenance (donations + a few disclosed affiliate links) — set it once and leave it.**

## 3. Focus / sequencing — if you only do 5 things in the next month

The highest-leverage path, balancing **credibility + discovery + low-maintenance money**. In order:

1. **Sequence-risk caveat + conservative-return stress-test toggle** (P0-1, P0-3) — fixes false certainty in ~½ day; the credibility floor that stops serious users bouncing.
2. **Request indexing + resubmit sitemap; confirm analytics baseline** (P0-2) — unblocks the entire SEO engine; nothing else ranks until this lands.
3. **Lean homepage copy into the wedge + cross-link the hub** (P0-5, P0-6) — sharpens positioning and lets authority flow, for copy-only effort.
4. **Stand up donations + 2–3 disclosed affiliate links + `/support` page** (P0-4, P0-7) — the whole low-maintenance monetization play, set once.
5. **Ship the first pre-65 healthcare guide + roll the SEO template onto one more calculator** (P1-1 start, P1-2 start) — begins the durable long-tail engine on our strongest, least-contested differentiator.

> Everything in #1–4 is days of work; #5 starts the multi-week content cadence. The Product Hunt launch and backlink sprint (P1-3, P1-5) come right after, once the homepage and first guides are solid.

## 4. Explicitly NOT doing / deprioritized

| Not doing (now) | One-line reason |
|---|---|
| Out-simulating cFIREsim/FIREproof feature-for-feature (VPW, Guyton-Klinger, Roth optimizer, glide paths) | A 14-year-head-start arms race we'd lose; wrong fight for a solo, low-maintenance founder. |
| Display / programmatic ads | Brand is *literally* "clean, ad-free"; tracking cookies break the privacy promise. Off the table permanently. |
| Paid Pro / freemium tier **now** | Paywalled features = ongoing build + support obligation; low expected willingness to pay. Parked until repeat power users visibly ask to pay. |
| B2B / white-label / embeds | Sales-driven, high-touch — the opposite of set-and-forget. Revisit only on unprompted inbound. |
| Sponsored newsletter | Requires running a newsletter first — the biggest ongoing-content commitment; parked. |
| Account-level tax engine **as a priority** | High value but high maintenance (annual tax tables); behind the low-maintenance credibility plays. |
| Chasing head terms ("retirement calculator", "fire calculator") | Owned by Vanguard/NerdWallet/Fidelity; a new YMYL domain won't crack page 1. Win the long-tail instead. |
| Forcing login to use core features | The no-login promise is a core differentiator; never gate the free workspace. |

## 5. Success metrics & milestones

Tie everything to the analytics already in place: **Google Search Console (GSC)** + **Vercel Web Analytics** (+ optional GA4 if added).

**Watch weekly (direction beats absolute numbers early):**

| Metric | Source | Early "good" |
|---|---|---|
| Indexed pages | GSC Coverage | All public + new guide/glossary pages indexed; no "discovered, not indexed" pileup |
| Impressions | GSC Performance | Trending up week-over-week as content ships; first long-tail terms appearing |
| Clicks + top queries | GSC | First clicks on ownable long-tail (household FIRE, FIRE for couples, pre-65 healthcare, SS break-even) |
| Visitors + top pages | Vercel | Guide/calculator pages (not just the home redirect) pulling traffic; launch-event bumps visible |
| Referrers | Vercel | Identifiable Reddit / Product Hunt / LinkedIn / blogger referrals — validates which channel works |
| Engagement | Vercel + product | Share running ≥1 strategy calc; "Help me choose" usage; repeat visits |
| Trust guardrail | Feedback form | Complaints about ads/advice stay near zero; donation conversions trickle in |

**Growth milestones (scenario B — content + launch/outreach burst):**

| Milestone | What "good" looks like | Visitors/mo |
|---|---|---|
| **Month 3** | All pages indexed; impressions rising; first long-tail queries; launch spikes banked; 1–2 backlinks landed | **~300–800** (mostly launch/referral) |
| **Month 6** | 20–35 pages aging; first page-1 long-tail rankings; steady small organic clicks; a couple of quality links live | **~1,000–2,500** |
| **Month 12** | 40–60 pages; several mid-tail terms on page 1; pre-65 healthcare content pulling its weight; organic now the compounding channel | **~3,000–6,000** — *5k/mo squarely in range* |

> Honest caveat (GROWTH §8): hitting the *top* of each band requires the backlink sprint and launch burst to actually land. Content-only, no link-building → expect the *bottom* of each band and a real risk of plateauing ~2k/mo into year two. The difference is a few weeks of front-loaded outreach, not years of grind.
